# Capacitor — Native iOS & Android Builds

SmartHabit uses [Capacitor](https://capacitorjs.com/) to wrap the SvelteKit PWA in a native
iOS and Android shell. The SvelteKit `adapter-static` build produces a `build/` directory
with a single `index.html` — this is the `webDir` that Capacitor serves inside the native
WebView.

## Prerequisites

| Platform | Requirement |
|----------|-------------|
| iOS      | macOS + [Xcode](https://developer.apple.com/xcode/) 15+ + Apple Developer account |
| Android  | [Android Studio](https://developer.android.com/studio) (any OS) |

Docker is **not** sufficient for native builds. Xcode and Android Studio must be installed
directly on the host machine.

## Initial setup (one-time per developer)

```bash
cd frontend

# Add iOS native project (macOS + Xcode required)
bunx cap add ios

# Add Android native project (Android Studio required)
bunx cap add android
```

The `ios/` and `android/` directories are git-ignored and must be generated locally.

## Dev workflow

```bash
# 1. Build the SvelteKit app
cd frontend && bun run build

# 2. Sync web assets + plugins into the native projects
bunx cap sync

# 3. Open Xcode (iOS) or Android Studio (Android)
bunx cap open ios
bunx cap open android
```

Or run steps 1+2 together:

```bash
cd frontend && bun run cap:build
```

### Live reload during development

To avoid rebuilding on every change, uncomment the `server.url` in
`frontend/capacitor.config.ts`:

```typescript
server: {
    url: 'http://localhost:5173',
    cleartext: true,
},
```

Then run the Vite dev server (`bun run dev`) and sync/open the native project. The native
WebView will load from the dev server instead of the bundled `build/` assets.

**Remember to comment these lines out before a production build.**

## Push notification setup per platform

### Web (PWA)
Web Push (VAPID) is registered automatically when the user opens the dashboard. No extra
setup needed — the service worker handles delivery.

### iOS (APNs)
1. In Xcode, enable **Push Notifications** under *Signing & Capabilities*.
2. Create an APNs key (p8) in the Apple Developer portal.
3. Add the key ID, team ID, and bundle ID to the backend `.env`:
   ```
   APNS_KEY_ID=...
   APNS_TEAM_ID=...
   APNS_BUNDLE_ID=com.smarthabit.app
   APNS_PRIVATE_KEY_PATH=/path/to/AuthKey.p8
   ```
4. The frontend registers automatically on first launch via `registerApnsPush()` in
   `src/lib/api/push-native.ts`.

### Android (ntfy)
1. The frontend generates a per-user ntfy topic (`smarthabit-user-{userId}`) and posts it
   to the backend.
2. The backend publishes notifications to that topic via the ntfy HTTP API.
3. The app subscribes to the SSE stream from `https://ntfy.sh/{topic}` for live delivery.

For production, host your own ntfy server and update `NTFY_SERVER` in
`src/lib/api/push-native.ts`.

## Project structure

```
frontend/
├── capacitor.config.ts       # Capacitor configuration
├── src/lib/
│   ├── platform.ts           # Platform detection (web / ios / android)
│   └── api/
│       ├── push.ts           # Public dispatcher — calls the right impl
│       ├── push-web.ts       # Web Push (VAPID) implementation
│       └── push-native.ts    # APNs (iOS) + ntfy (Android) implementation
├── ios/                      # Generated by cap add ios  (git-ignored)
└── android/                  # Generated by cap add android (git-ignored)
```

## Useful commands

| Command | Description |
|---------|-------------|
| `bun run cap:build` | Build SvelteKit + sync to native |
| `bun run cap:sync`  | Sync web assets only (no build) |
| `bun run cap:ios`   | Open Xcode |
| `bun run cap:android` | Open Android Studio |
